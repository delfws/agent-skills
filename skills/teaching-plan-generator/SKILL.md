# 教案生成技能 (Teaching Plan Generator)

## 概述
根据用户提供的课文内容（文字或图片），自动生成结构完整、风格规范的教案。适用于中小学各学科，尤其是音乐、语文等需要赏析类教案的场景。

## 触发条件
当用户发送课文图片/文字并要求生成教案、教学设计、备课内容时触发此技能。

---

## 教案结构模板

生成的教案必须包含以下完整结构：

### 1. 表头信息
```
课题：[课文标题]
课时：[建议课时数]
课型：[新授/复习/欣赏/实践]
备课时间：[日期]
```

### 2. 教学目标（3条，逐层递进）
按以下三个维度撰写：
- **感受与体验：** 让学生感受/体会课文的核心情感或意境
- **能力与方法：** 培养学生的某项具体能力（如聆听、演唱、分析等）
- **拓展与认知：** 拓展学生的文化/知识视野

每条目标使用动词开头：感受、体会、了解、认识、掌握、培养、激发、萌发

### 3. 教学分析
- **重点：** 本课最核心的教学内容（通常1-2个要点）
- **难点：** 学生理解或掌握有困难的部分
- **教法：** 具体教学方法（如情境教学法、聆听模唱法、讲授法、讨论法等）
- **教具：** 需要的教学资源（如多媒体课件、钢琴、音频等）

### 4. 教学过程（四栏表格）

使用表格格式，包含四列：

| 教学步骤 | 教师活动 | 学生活动 | 设计意图 |
|---------|---------|---------|---------|

**教学步骤的标准流程：**

#### 步骤一：导入（1-2个环节）
- 用与课文主题相关的情境引入（视频、音频、图片、故事、提问等）
- 引出课题，激发兴趣
- 设计意图关键词：创设情境、激发兴趣、引出主题

#### 步骤二：初感/整体感知
- 完整呈现课文内容（播放、朗读等）
- 引导学生关注整体感受（速度、情绪、意境等）
- 设计意图关键词：建立整体印象、初步感知

#### 步骤三：深入赏析/学唱/分析
- 分析课文的具体元素（歌词/文字、旋律、结构等）
- 逐句/逐段讲解和练习
- 设计意图关键词：深入理解、突破重点、掌握技能

#### 步骤四：拓展延伸
- 介绍相关文化背景、作者信息、历史典故等
- 对比不同版本/演绎方式（如有）
- 设计意图关键词：拓展视野、深化理解、文化认知

#### 步骤五：课堂小结
- 引导学生回顾本课收获
- 总结核心情感/知识点
- 设计意图关键词：巩固记忆、情感升华

### 5. 板书设计
用简洁的关键词呈现本课核心内容：
```
[课文标题]（[体裁/类型]）
[要素1]：[关键词]
[要素2]：[关键词]
[核心情感/主题]：[关键词]
```

### 6. 课堂小结（文字版）
用1-2句话总结本课学习内容和期望：
> "今天我们[学习/聆听/赏析]了……，感受了……，希望大家能……"

### 7. 教学反思
简要记录教学中的不足或改进方向：
> "部分学生[具体问题]，对[某方面]的理解不够深入。"

## 写作风格规范

### 教学过程表格内容要求（重要）
- **教师活动、学生活动、设计意图三栏内容要精简但有内容，每栏1-2句话**
- 不要写成长段落，保持简洁明了

### 语言特点
- 使用精练的教学语言，多用四字短语（如"中速稍慢""抒情悠远""创设情境"）
- 教师活动要具体到可操作的行为（如"播放视频""出示歌词""提问"）
- 学生活动简洁明确（如"安静聆听""圈画景物""跟琴模唱"）
- 设计意图用高度概括的短语（如"快速创设草原情境，引出主题"）

### 教师活动示例
- "播放XX视频/音频，提问：'听/看后有什么感受？'"
- "出示XX内容，引导学生观察/思考"
- "范唱/范读XX片段"
- "用XX方式带练/示范"

### 学生活动示例
- "安静聆听，感受XX"
- "自由朗读，圈画XX"
- "跟琴模唱，分句学唱"
- "小组讨论，分享感受"

### 设计意图示例
- "快速创设XX情境，引出主题"
- "建立歌曲/课文整体印象"
- "实践中掌握XX，突破教学重点"
- "拓展XX认知，深化情感体验"

## 输出格式

### 文字形式（默认）
直接输出完整教案文本，使用 Markdown 格式，表格部分使用 Markdown 表格语法。

### 图片形式（用户要求时）
当用户要求图片输出时，使用 SVG + sharp 生成教案图片。

#### 依赖
- **sharp**: 安装在技能包目录的 `node_modules/sharp`
- **生成脚本**: 技能包目录下的 `scripts/generate-image.js`

#### 方式一：使用生成脚本（推荐）

1. 先生成完整的文字教案内容
2. 将教案数据整理为 JSON 格式，写入临时文件
3. 运行脚本生成图片：
```bash
node scripts/generate-image.js /tmp/lesson-data.json
```

**JSON 数据格式：**
```json
{
  "title": "小学音乐教案",
  "outputPath": "/tmp/教案-歌曲名.png",
  "infoTable": [
    ["课 题", "歌曲名称"],
    ["课 时", "1课时（40分钟）"],
    ["课 型", "唱歌课"],
    ["备课时间", "2026年6月9日"]
  ],
  "sections": [
    {
      "title": "一、教学目标",
      "type": "list",
      "items": [
        { "label": "感受与体验", "text": "目标内容..." },
        { "label": "能力与方法", "text": "目标内容..." }
      ]
    },
    {
      "title": "二、教学分析",
      "type": "subsections",
      "subsections": [
        {
          "title": "教学重点",
          "items": [{ "text": "重点内容..." }]
        }
      ]
    },
    {
      "title": "三、教学过程",
      "type": "table",
      "headers": ["教学步骤", "教师活动", "学生活动", "设计意图"],
      "colWidths": [120, 260, 170, 150],
      "rows": [
        {
          "cells": [
            { "text": "一、情境导入", "bold": true },
            { "text": "教师活动内容..." },
            { "text": "学生活动内容..." },
            { "text": "设计意图..." }
          ]
        }
      ]
    },
    {
      "title": "四、板书设计",
      "type": "board",
      "content": "板书内容（用换行分隔）"
    },
    {
      "title": "五、课堂小结",
      "type": "text",
      "content": "小结文字..."
    },
    {
      "title": "六、教学反思",
      "type": "list",
      "items": [{ "text": "反思内容..." }]
    }
  ]
}
```

**sections.type 说明：**
- `list`: 列表型内容，支持 `label`（加粗标题）+ `text`（正文）
- `subsections`: 带子标题的分组内容（如教学分析的重点/难点/教法），子标题加粗显示，条目按层级缩进（子标题 75px → 条目 100px → 正文 110px）
- `table`: 四栏表格（教学过程），需定义 `headers` 和 `colWidths`，单元格内容自动垂直居中
- `board`: 板书设计，高度自适应内容行数，`content` 用换行符分隔各行
- `text`: 普通段落文本

#### 图片生成已知优化（2026-06-09）

脚本经过多轮调试，已解决以下问题：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 中文文字溢出单元格 | 字符宽度计算使用固定 0.55 倍率 | 改为逐字符计算：中文 1.0 倍、英文 0.6 倍 |
| 文字在词语中间断行 | 无标点优先断行逻辑 | 优先在标点符号后断行，避免硬切词语 |
| 板书容器文字溢出重叠 | 容器高度固定 280px | 改为动态计算：行数 × 行高 + 内边距 |
| 教学分析全段加粗 | 标签和正文在同一行渲染 | 有标签时分两行：标签加粗 + 正文正常 |
| 表格内容顶部堆叠 | 无垂直居中 | 按实际行高计算居中偏移 |
| 子标题视觉层次弱 | 纯文字无装饰 | 加蓝色竖条 + 蓝色圆点标记 |
| 第二部分缩进不对齐 | 子标题和第一部分条目缩进不一致 | 子标题 75px，条目 100px，正文 110px（与第一部分条目对齐） |
| 第二部分装饰过多 | 蓝色竖条和圆点喧宾夺主 | 去掉蓝色竖条和圆点，靠字体粗细+缩进区分层次 |

#### 间距配置（SPACING 常量）

脚本顶部 `SPACING` 对象控制所有间距，可按需调整：
```js
const SPACING = {
  pagePaddingX: 55,        // 页面左右内边距
  pagePaddingTop: 50,      // 页面顶部内边距
  sectionGap: 48,          // 板块间距 (h2 之间)
  subsectionGap: 30,       // 子板块间距 (h3 之间)
  listItemGap: 18,         // 列表项间距
  tableCellPadX: 14,       // 表格单元格水平内边距
  tableCellPadY: 12,       // 表格单元格垂直内边距
  tableRowMinHeight: 40,   // 表格行最小高度
  lineGap: 6,              // 同一段落内行间距
  paragraphGap: 16,        // 段落间距
};
```

#### subsections 类型缩进配置

脚本中 `subsections` 渲染的缩进位置：
```js
const subTitleX = SPACING.pagePaddingX + 20;  // 子标题：75px（与 list 类型条目对齐）
const subItemX = SPACING.pagePaddingX + 40;  // 条目内容：100px
// 正文续行：subItemX + 10 = 110px
```

调整原则：
- 子标题缩进应与 `list` 类型的条目缩进一致（75px）
- 条目内容再缩进 25px，形成清晰层次
- 正文续行比条目再缩进 10px
- 不使用蓝色竖条或圆点装饰，靠字体粗细和缩进区分层次

#### 方式二：直接使用脚本生成 SVG

如果需要更灵活的控制，可以参考脚本中的 `generateSVG()` 函数，直接构造 SVG 字符串。

#### 发送图片

将生成的教案图片发送给用户即可。具体发送方式取决于使用的 Agent 平台。

## 课文内容提取

当用户上传课文图片时：
1. 使用 omni（bash mimo_api.sh）提取图片中的文字内容
2. 识别课文标题、体裁、正文、注释等信息
3. 基于提取的内容生成教案

## 注意事项

1. **紧贴课文：** 教案内容必须紧密围绕具体课文，不能泛泛而谈
2. **因材施教：** 根据课文类型（音乐/语文/数学等）调整教学过程的侧重点
3. **情感导向：** 教案要体现对课文情感/意境的深度理解
4. **可操作性：** 教师活动必须是课堂上可以实际执行的具体行为
5. **逻辑连贯：** 各教学环节之间要有自然的过渡和递进关系
6. **图片间距：** 生成脚本中的 SPACING 配置控制间距，如需调整可修改脚本顶部的常量
7. **sharp 路径：** sharp 安装在技能包目录的 `node_modules/sharp`，脚本通过 `__dirname` 相对路径引用
8. **中英文混排：** 脚本已支持中英文字符宽度自动识别，中文字符按 1.0 倍字体大小计算，英文按 0.6 倍
9. **JSON 生成：** 建议用 Python `json.dump()` 生成 JSON 文件，避免手写 JSON 时中文引号导致解析错误
10. **图片发送：** 生成的图片直接发送给用户即可
11. **board 类型高度：** 板书设计容器高度已改为动态计算，无需手动指定高度
